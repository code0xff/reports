Visa, Mastercard의 AI Agent 결제 시장 접근 방식 및 구현

초록

Visa와 Mastercard는 2025년 4월 각각 Visa Intelligent Commerce와 Mastercard Agent Pay를 발표하며, AI 에이전트가 사용자 대신 결제까지 수행하는 시장을 본격적으로 겨냥했다.[1][2] 2026년 5월 현재 두 회사의 공통 전략은 기존 카드 네트워크를 대체하는 것이 아니라, 네트워크가 이미 보유한 토큰화·인증·분쟁/사기관리 역량을 에이전트 상거래로 확장하는 데 있다.[1][2][3] 차이는 구현 중심축에서 나타난다. Visa는 Intelligent Commerce Connect, Agentic Ready, Trusted Agent Protocol로 생태계 온보딩과 상호운용 레이어를 강조하고,[3][4][5] Mastercard는 Agentic Tokens와 Acceptance Framework를 통해 네트워크 내 신뢰된 에이전트 결제 식별·수용을 전면에 둔다.[2][6]

1) 시장 배경: 에이전틱 커머스 전환

Visa는 2025년 4월 30일 Intelligent Commerce를 발표했고,[1] Mastercard는 하루 앞선 4월 29일 Agent Pay를 발표했다.[2] 두 발표 모두 "AI 에이전트가 탐색, 선택, 결제 수행"을 핵심 시나리오로 제시한다.[1][2] 독립 보도도 이를 동일하게 해석하며, 양사의 행보를 AI 쇼핑 결제 표준 경쟁의 시작으로 봤다.[3]

2) Visa 접근법: 포트폴리오형 확장

Visa의 접근은 단일 제품보다 연속된 포트폴리오 확장에 가깝다.

2025년 4월 Intelligent Commerce 출범 시점에 Visa는 토큰화 기반 AI-Ready Cards, 소비자 한도/조건 통제, 실시간 신호 기반 통제 구조를 제시했다.[1] 이후 2025년 10월에는 Visa와 Cloudflare가 Trusted Agent Protocol(TAP)을 공개해 에이전트-머천트 신뢰 검증 프레임을 별도로 제시했다.[4][5] 2026년 4월에는 Intelligent Commerce Connect(ICC)를 출시해 Visa Acceptance Platform 단일 통합으로 결제개시·토큰화·인증·지출통제를 제공하고, 비자 외 카드 네트워크 API까지 연계하는 온램프 모델을 발표했다.[6]

같은 달(2026년 4월 29일) Visa는 Agentic Ready를 유럽에서 APAC/LATAM으로 확장하며, 발급사·파트너가 실거래형 통제환경에서 등록/토큰화/인증/승인 흐름을 검증하도록 프로그램화했다.[7] 이 구조는 "표준 제시 + 파트너 준비도 관리"를 병행하는 방식이다.

3) Mastercard 접근법: Agent Pay + Agentic Tokens

Mastercard는 Agent Pay를 "에이전트 결제 프로그램"으로 정의하면서, Agentic Tokens를 핵심 자격증명 계층으로 제시했다.[2] 또한 신뢰된 에이전트의 등록/검증을 결제 이전 단계의 필수 조건으로 두고, 소비자 통제·인증·사기방지·분쟁지원을 포함한 end-to-end 보안 서사를 함께 제시했다.[2]

2025년 10월 PayPal과의 파트너십 발표에서는 Agent Pay를 대형 지갑 생태계에 결합하고, PayPal이 Mastercard Agent Pay Acceptance Framework를 파일럿한다고 밝혔다.[8] 같은 해 12월 LATAM 발표에서는 해당 프레임워크가 거래 전 agent registration and verification을 요구한다고 명시했다.[9] 2026년 1월 APAC 발표에서는 호주 내 인증된 agentic 거래 사례를 공개하며 지역 확장을 공식화했다.[10]

4) 구현 아키텍처 비교

두 회사의 공통점은 명확하다. 1) 사용자 대신 행동하는 에이전트에 대해 별도 신뢰/인증 계층을 둔다.[1][2][4][9] 2) 원시 카드정보 대신 토큰화 자격증명을 중심에 둔다.[1][2][6] 3) 소비자 통제(한도/조건/권한)와 분쟁 가능성 대응을 설계에 포함한다.[1][2]

차이점도 분명하다.

  • Visa: ICC를 통해 수용(acceptance) 측 통합 마찰을 줄이는 허브형 메시지(단일 통합, 다중 네트워크/토큰볼트 호환)를 강하게 낸다.[6]
  • Mastercard: Agentic Tokens와 Acceptance Framework로 네트워크 내부의 거래 식별·검증·수용 규율을 명시적으로 내세운다.[2][9]

즉, Visa는 생태계 연결성과 온보딩 경로를, Mastercard는 네트워크 레벨 통제 프레임을 상대적으로 전면에 배치하고 있다. _(해석)_ [6][9]

5) 트랙션과 확장 현황 (2025~2026)

Visa는 2025년 12월 기준으로 수백 건의 secure agent-initiated transaction, 100+ 파트너, 20+ 직접 통합 에이전트/인에이블러를 공개했다.[11] 2026년 4월에는 비즈니스 온램프(ICC)와 지역 확장(Agentic Ready)을 연달아 발표했다.[6][7]

Mastercard는 2025년 4월 출시 후, 2025년 10월 PayPal 연계, 2025년 12월 LATAM, 2026년 1월 APAC 사례를 발표하며 지역·파트너 확장형 로드맵을 제시하고 있다.[8][9][10]

다만 공개된 수치는 벤더 발표 중심이며, 승인율·사기율·차지백 등 핵심 운영 KPI의 제3자 검증 비교는 제한적이다. _(vendor-stated)_ [11][9][10]

6) 리스크와 실행 시사점

핵심 리스크는 "누가 어떤 의도로 결제를 실행했는지"를 기술·운영·법적 관점에서 일치시키는 문제다. Visa는 TAP과 Agentic Ready로 신뢰 신호 교환 및 준비도 검증을 제시하고,[7][4] Mastercard는 Acceptance Framework와 Agentic Token 기반 인증 흐름으로 대응한다.[2][9]

실행 우선순위는 다음과 같다.

  • 발급사/은행: 에이전트 권한 정책(한도/카테고리/시간/사용자 인증)을 카드 리스크 정책과 결합.
  • 머천트/PSP: 에이전트 식별, 허용 정책, 감사로그/분쟁증적 수집을 체크아웃 경로에 내장.
  • 에이전트 빌더: 토큰화 자격증명 사용, 사용자 통제 UI, 거래 가시성(누가/언제/무엇을 결제했는지) 확보.

위 3가지는 양사 프로그램의 공통 요구사항에서 직접 도출된다. _(해석)_ [1][2][6][9]

7) 결론

2026년 5월 시점에서 Visa와 Mastercard의 AI 에이전트 결제 전략은 "카드 네트워크 신뢰 모델의 에이전트 확장"이라는 공통 축 위에 있다. Visa는 연계 허브·준비도 프로그램 중심으로 외연을 넓히고,[6][7] Mastercard는 Agentic Token/Acceptance Framework 중심으로 네트워크 통제를 구조화한다.[2][9] 최종 승부는 기술 우위 단독이 아니라, 발급사·가맹점·에이전트 플랫폼이 실제로 낮은 마찰로 연결되는 운영 표준 경쟁에서 갈릴 가능성이 높다. _(early signal)_ [3][5]
