# Uncertainties

- `alloy-rs`를 Java로 래핑한 성숙한 공개 사례가 많지 않으므로, 제품화 난이도와 성능 비용 추정에는 해석이 포함된다.
- `web3j`와 `alloy-rs`의 비교는 역할이 완전히 같은 두 라이브러리의 직접 성능 비교가 아니라, 공개 문서와 구조를 기준으로 한 범위·현대성 비교다.
- `FFM/jextract`의 실무 매력은 크지만, 조직의 JDK baseline이 22 이상인지에 따라 현실성이 크게 달라진다.
- `alloy-rs`의 공식 성능 수치는 Rust 네이티브 문맥 기준이므로, Java wrapper 경유 시 동일 수치를 기대할 수는 없다.
